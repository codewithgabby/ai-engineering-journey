# Engineering Review and Systems Thinking

## Backend Engineering Is More Than Writing Code

This week helped me realize that backend engineering is deeper than just writing Python syntax.

A backend engineer must think about:

- memory
- scalability
- performance
- architecture
- system behavior
- production reliability

---

## Generators and Scalability

Generators are important because they help backend systems process data little by little instead of loading everything into memory at once.

This is important for:

- AI systems
- streaming systems
- dashboards
- analytics systems
- uploaded files
- large datasets

Generators help reduce memory usage and improve scalability.

---

## Async Programming and Performance

Async programming helps backend systems avoid blocking.

Instead of waiting for one request to finish completely before another request starts, async systems allow other tasks to continue while waiting.

This is why FastAPI is fast.

Async systems are important for:

- APIs
- AI systems
- notifications
- real-time systems
- dashboards
- external API calls

---

## Decorators and Clean Architecture

Decorators help engineers avoid repeating code everywhere.

They can be used for:

- logging
- authentication
- monitoring
- permissions
- timing execution
- rate limiting

Decorators help backend systems become cleaner and easier to maintain.

---

## Mutability and Backend Reliability

Mutable objects can create hidden bugs in backend systems if multiple parts of the application modify the same object unexpectedly.

Understanding references and mutable state is important in:

- async systems
- caching
- shared backend state
- production systems

---

## Biggest Mindset Shift

One major realization this week is that backend engineering is not only about making code work.

It is also about understanding:

- how systems behave
- how memory works
- how APIs scale
- how backend systems become reliable
- how engineers debug production systems

I’m beginning to think more like an engineer instead of only focusing on syntax.
